# Legal And Packaging Policy

This repository publishes original skill files written by the maintainer and contributors. Those files describe reusable engineering methods in abstract, instructional form.

## Scope Of Rights

The repository license applies only to original materials authored for this repository, including:
- skill rewrites
- sanitized public metadata
- indexes
- install helpers
- documentation

It does not grant rights in any third-party source code, documentation, tests, prompts, trademarks, product names, logos, or proprietary assets that may have been reviewed during the authoring process.

## No Source Redistribution

Public artifacts in this repo must not contain:
- source code excerpts
- copied comments or docstrings
- copied tests
- copied prompts
- copied ASCII art, logos, or proprietary UI assets
- mirrored internal file trees or raw local paths

## Abstraction Standard

Allowed public content:
- behavior-first summaries
- method-level workflows
- generalized constraints and heuristics
- high-level examples rewritten in original language

Disallowed public content:
- line-for-line paraphrases of protected code comments
- raw repository path mirrors
- long chains of internal function names or private identifiers where generic wording would work
- content that allows a reader to reconstruct proprietary code structure instead of learning the transferable method

## Public Vs Private Provenance

This public repo uses only coarse provenance such as pattern family, subsystem category, review status, and bundle membership.

Raw provenance such as exact source repositories, internal mirror paths, or local absolute paths must stay outside the public distribution repo.

## Trademark And Brand Usage

Third-party product names may appear only to describe compatibility or nominative reference.

This repository is independent and is not affiliated with, endorsed by, sponsored by, or certified by Anthropic, OpenAI, or any other third-party vendor named here.

All third-party product names, logos, and trademarks remain the property of their respective owners.

## Removal And Correction Process

If a public artifact appears too close to protected upstream material:
1. report the file and concern to the maintainer
2. the maintainer reviews the artifact against the abstraction standard
3. the file is rewritten, replaced, or removed if needed
4. the public index is updated to reflect the correction

This file is packaging guidance, not legal advice.
